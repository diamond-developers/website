Error code 429.

What is this?

By: Era

What is error 429?

Musical uses the Youtube Data V3 API to play content. The Youtube Data API has "interesting" rate limits and will often rate limit Musical, causing error 429. However it displays as a 404 (not found) error, when the song is found. Thats because the bot cant index the Youtube API to search for that song, causing the faulty 404 error.

How can I fix it?

Unfortunately, you cant fix this error. The only way to fix it, would be to restart the bot, which is locked to myself (era) and Spxter. However you can contact us in our support server, and ask for us to restart the bot.

Is this fixed?

We released a patch yesterday (25/10/2020) that looks like it has fixed this error. The fix is not a real fix, because we have no control over the Youtube rate limits.

How do I avoid this error?

Musical has to rely on you to help us prevent this issue. The best thing you could do is wait a minute until adding a song to your playlist, or creating a youtube playlist with all of your songs, and just play it once.
If you need to replay your song, please the loop feature (m!loop), and do not constantly play it again.
